Understanding CSRF Mechanics

                
CSRF attacks exploit the automatic inclusion of authentication credentials (cookies, session tokens) with every browser request to a domain.

                
                
    
                    
  1. User Authentication: Victim logs into legitimate website (bank.com)
    2. 
                    
  2. Session Establishment: Browser stores authentication cookie
    3. 
                    
  3. Attacker Trap: Victim visits malicious site (evil.com)
    4. 
                    
  4. Cross-Site Request: Malicious page submits form to bank.com
    5. 
                    
  5. Automatic Authentication: Browser includes bank.com cookies
    6. 
                    
  6. Unauthorized Action: Bank processes request as legitimate
    7. 
                

            


            

                
Vulnerable Code Examples

                
                
❌ Vulnerable Implementation

                

                    // No CSRF protection

                    app.post('/api/transfer-funds', authenticate, (req, res) => {

                      const { recipient, amount } = req.body;

                      transferMoney(req.user.id, recipient, amount);

                      res.json({ success: true });

                    });
                


                
✅ Secure Implementation

                

                    // With CSRF token validation

                    app.post('/api/transfer-funds', [authenticate, csrfProtection], (req, res) => {

                      const { recipient, amount, _csrf } = req.body;

                      if (!validateCsrfToken(req.session.csrfSecret, _csrf)) {

                        return res.status(403).json({ error: 'Invalid CSRF token' });

                      }

                      transferMoney(req.user.id, recipient, amount);

                      res.json({ success: true });

                    });
                

            


            

                
Defense Mechanisms

                

                    

                        
🎫 Synchronizer Tokens

                        
Server generates unique token per session, validated on submission

                    

                    

                        
🍪 SameSite Cookies

                        
Restrict cookie transmission to same-site requests only

                    

                    

                        
🔍 Origin Validation

                        
Verify Origin/Referer headers match expected domain

                    

                    

                        
🔐 Double Submit

                        
Token in both cookie and request parameter must match

                    

                    

                        
📋 Custom Headers

                        
Require custom headers that CSRF cannot forge

                    

                    

                        
🔑 Re-authentication

                        
Require password confirmation for sensitive operations
